# Harmonium 采样文件获取清单

## 目标

收集一批可以合法用于网站的 harmonium 采样文件，用于网页键盘播放。

目标结果：
- 准备好 13 个单音文件
- 确认每个文件都具备可商用或明确可用的授权状态
- 保存来源和授权证据
- 文件可以直接重命名并接入当前项目

## 必需音名

你需要为下面 13 个音各准备 1 个可用文件：
- `c4`
- `csharp4`
- `d4`
- `dsharp4`
- `e4`
- `f4`
- `fsharp4`
- `g4`
- `gsharp4`
- `a4`
- `asharp4`
- `b4`
- `c5`

## 推荐来源优先级

### 方案 A：自己录音，或找演奏者定制录音
这是长期最稳的方案，版权最干净。

要求：
- 一个音一个文件
- 尽量录干声
- 每个音持续 2 到 5 秒
- 整套音都用同一台 harmonium、同一支麦克风、同一录音位置
- 不要带伴奏、节奏、无人声哼唱

需要额外保存：
- 演奏者姓名
- 录制日期
- 授权说明、聊天记录、合同或发票
- 能证明你可以在网站和商业项目中使用这些音频的证据

### 方案 B：购买 sample pack
这是最快能上线的商业化方案。

搜索关键词建议：
- `harmonium one shot`
- `harmonium single note`
- `harmonium sample pack`
- `harmonium dry sample`

只选明确允许商业使用的包。

以下情况不要买：
- 只有 loop，没有单音
- 只有完整旋律或乐句，没有 isolated note
- License 写得不清楚
- 混响太重、噪音太大、音头太慢

需要额外保存：
- 商品链接
- 发票或订单记录
- 购买当天的 license 页面截图或 PDF

### 方案 C：免费素材库先做 MVP
这个方案适合先验证需求，不建议长期正式使用。

使用规则：
- 优先使用 `CC0`
- `CC-BY` 可以用，但你要准备好按要求署名
- 不要用 `CC-BY-NC` 做商业项目
- 不要用 license 不明确的文件

需要额外保存：
- 来源页面链接
- 作者或上传者名称
- license 类型
- 下载当天的页面截图

## 音频质量检查清单

选择文件前，确认这些条件：
- 起音快，不拖沓
- 音高稳定
- 是单独的一个音，不是旋律片段
- 没有明显环境噪音、底噪、敲击声串入
- 整组文件的音量尽量接近

## 文件处理清单

下载或录音完成后：
1. 裁掉开头空白。
2. 保留自然衰减尾音。
3. 尽量统一各个音的音量。
4. 原始母带文件单独备份。
5. 如有需要，导出网页使用的 `.mp3` 版本。

## 项目内放置位置

最终网页可直接使用的文件放到这里：
- `public/audio/harmonium/`

支持格式：
- `.mp3`
- `.wav`
- `.ogg`

## 建议的临时命名方式

如果来源文件命名很乱，先不要自己硬猜。

建议先放到项目外的临时目录，例如：
- `downloads/harmonium-samples/raw/`
- `downloads/harmonium-samples/selected/`

等你把选好的文件列表发给我后，我再帮你统一映射成项目需要的音名。

## 你拿到文件后发给我什么

你准备好文件后，发我下面任意一种即可：
- 文件所在文件夹路径
- 选中文件名的截图
- 一份文本列表，例如：
  - `C4_take1.wav`
  - `Db4_clean.wav`
  - `D4_clean.wav`
  - ...

我接下来可以继续帮你做：
- 检查是否缺音
- 批量重命名成项目要求的 13 个音名
- 移动到 `public/audio/harmonium/`
- 跑构建检查

## 明确不要用的来源

不要使用：
- 直接从 YouTube 抠下来的音频
- 从别人的产品或网站里直接扒下来的音频
- GitHub 仓库里没有明确授权的音频
- `CC-BY-NC` 素材去做商业站
- 不是单音而是整段旋律的文件
